# 🔐🚪 yourorder-auth-gateway

## 📌 О проекте

**yourorder-auth-gateway** - это первый сервис системы YourOrder.

Он объединяет в себе:

-   🔐 Аутентификацию и авторизацию пользователей\
-   🚪 API Gateway (единую точку входа в систему)\
-   🔑 JWT (Access + Refresh токены)\
-   🔄 Проксирование запросов в другие микросервисы

> ⚠️ Сейчас это архитектурное описание проекта. Реализация будет
> добавлена позже.

------------------------------------------------------------------------

## 🏗 Роль в архитектуре

    Client
       ↓
    Auth + Gateway
       ↓
    Core / Integration / Async

Клиент обращается только к этому сервису.\
Все остальные микросервисы недоступны напрямую.

------------------------------------------------------------------------

# 🔐 AUTH часть

## 🎯 Назначение

-   Регистрация пользователей\
-   Проверка логина и пароля\
-   Генерация JWT токенов\
-   Управление Refresh токенами\
-   Работа с ролями

------------------------------------------------------------------------

## 📦 Основные сущности

### User

-   id (UUID)
-   username (уникальный)
-   email (уникальный)
-   password (BCrypt)
-   role (ADMIN / SUPPLIER / RETAILER)
-   tenantId
-   createdAt

### RefreshToken

-   id
-   userId
-   token (UUID)
-   expiresAt
-   revoked

------------------------------------------------------------------------

## 🌍 Основные маршруты

### POST `/auth/register`

``` json
{
  "username": "alex_92",
  "email": "alex@mail.com",
  "password": "123456",
  "role": "RETAILER"
}
```

------------------------------------------------------------------------

### POST `/auth/login`

``` json
{
  "login": "alex_92",
  "password": "123456"
}
```

Ответ:

``` json
{
  "accessToken": "...",
  "refreshToken": "..."
}
```

------------------------------------------------------------------------

### POST `/auth/refresh`

``` json
{
  "refreshToken": "uuid"
}
```

------------------------------------------------------------------------

### POST `/auth/logout`

Выход пользователя (отзыв refresh токена)

------------------------------------------------------------------------

# 🔑 JWT модель

## Алгоритм

Используется **RS256 (RSA)**:

-   Private key - хранится внутри сервиса
-   Public key - используется для проверки подписи

------------------------------------------------------------------------

## 🟢 Access Token

-   Живёт ~15 минут
-   Stateless
-   Не хранится в БД

Пример payload:

``` json
{
  "sub": "userId",
  "username": "alex_92",
  "role": "RETAILER",
  "tenantId": "123",
  "iat": 1710000000,
  "exp": 1710000900
}
```

------------------------------------------------------------------------

## 🔵 Refresh Token

-   UUID
-   Хранится в БД
-   Живёт \~7 дней
-   Может быть отозван

------------------------------------------------------------------------

# 🚪 GATEWAY часть

## 🎯 Назначение

-   Проверка JWT
-   Проверка срока действия токена
-   Проверка ролей
-   Проксирование запросов
-   Добавление служебных headers

------------------------------------------------------------------------

## 📍 Маршрутизация

  Route                   Назначение
  ----------------------- ---------------------
  `/auth/**`              Публичные endpoints
  `/api/core/**`          Core Service
  `/api/integration/**`   Integration Service
  `/api/async/**`         Async Service

------------------------------------------------------------------------

## 🔄 Обработка запроса

1.  Проверяется заголовок `Authorization: Bearer`
2.  Валидируется подпись JWT
3.  Проверяется срок действия
4.  Извлекаются данные пользователя
5.  В запрос добавляются headers:

```{=html}
<!-- -->
```
    X-User-Id
    X-Username
    X-User-Role
    X-Tenant-Id

6.  Запрос проксируется в нужный сервис

------------------------------------------------------------------------

# ❌ Ошибки

| Ситуация              | Код |
|-----------------------|-----|
| Нет токена            | 401 |
| Неверный токен        | 401 |
| Истёкший токен        | 401 |
| Недостаточно прав     | 403 |

------------------------------------------------------------------------

# 🔐 Безопасность

-   Пароли хранятся в BCrypt
-   Access token не хранится в БД
-   Refresh token хранится в БД
-   Private RSA ключ не хранится в Git
-   Остальные сервисы недоступны напрямую
